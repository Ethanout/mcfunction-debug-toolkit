# 严格审计整改计划

[English](HOSTILE_AUDIT_REMEDIATION.md) | **简体中文**

该文件记录了独立的敌意审查以及之后做出的决定
根据当前 Minecraft 26.2 实施检查每项声明。保留它
在修复过程中进行更新。

## 接受 P1 调查结果

- [x] 绑定查询与 256 个渲染叶预算分开工作。限制
  实体/分数源访问量和漂亮组件之前选定的总 NBT 字节数
  转换。缺失的路径不得允许无限制的每实体 NBT 工作。
- [x] 用显式回调替换last-callback-wins命令结果，
  成功/失败和结果总和聚合。保留未报告的结果
  状态。
- [x] 在排队服务器工作开始之前添加截止时间/取消门。曾经
  工作已经开始，报告 `unknown_outcome` 而不是暗示 504 意味着
  命令没有执行；停止命令之间的超时批处理。
- [x] 按计数和响应字节进行页面调试事件、诊断和客户端聊天
  预算。绑定捕获的命令反馈并暴露截断/更多/最新
  元数据。
- [x] 将渲染线程屏幕截图工作保留到帧缓冲区捕获。移动像素
  转换/JPEG 编码到一个有界工作人员，编码一次，强制执行一次
  进行中的请求和 JPEG 字节限制，并使用无冲突的文件名。
- [x] 将手工维护的中文路径自检列表替换为JUnit平台
  发现，因此每个 `@Test` 都由 `gradlew check` 执行。

## 已接受的 P2/P3 调查结果

- [x] 在实时注册表之外暂存重新加载指令并自动交换
  仅在成功完成时才生成不可变的生成。重叠香草
  重新加载入口点仍然需要单独的真实游戏确认。
- [x] 不要将整数 `d` 格式应用于小数/非有限值；保留
  原始的本机值而不是默默地截断它。
- [x] 仅在输入步骤的持续时间/操作完成后对其进行计数，而不是在输入步骤完成时进行计数
  开始。继续取消索赔已完成的有效保留。
- [x] 将真实数据包移至版本化测试数据中并提供实时数据包
  断言脚本；被忽略的本地保存和散文记录不是
  可重复的回归测试。
- [x] 绑定运行时诊断速率限制密钥缓存并在某个时间清除它
  适当的生命周期边界。

## 拒绝或故意推迟索赔

- `.0g` 不被拒绝：Python 本身将精度 0 视为有效值
  数字。定制 `.0p` 遵循相同的规则。该行为必须被记录并
  测试而不是默默地改变。
- `DangerPolicy` 只是一个确定性的意外误用防护装置。它无法停止
  嵌套 `execute run`、数据包函数或其他间接副作用以及
  绝不能被描述为授权/安全边界。一个真正的沙盒
  需要受限的命令源/调度程序，并且超出了范围
  这个正则表达式列表。
- 如果没有可重现的陈旧深度，任何函数堆栈重写都是不合理的
  失败。
- 真正重叠或异常失败的 Minecraft 重新加载仍然是一个问题
  真实游戏验证项；不要仅从综合单元测试中声明它。
